# コンカフェおじさん落ちものパズル 企画仕様書

Version: 0.2 prototype  
Last updated: 2026-04-26  
Target: browser / PC / mobile

## 1. 企画概要

「コンカフェおじさん落ちものパズル」は、コンカフェの常連視点を起点にした落ちものパズルゲームです。プレイヤーは営業中のフロアを管理する視点で、上から来店する「おじさん客」を下段のキャスト列へ配置し、満足させて帰していきます。

通常の色合わせパズルではなく、客とキャストの「推し」「指名」「推し被り」「空気の悪化」「常連化」といった関係性を、パズルの判断材料として扱います。

## 2. コンセプト

### 2.1 コアコピー

彼女たちは笑顔を作り、私はおじさんを配置する。

### 2.2 目指す体験

- コンカフェ文化の感情経済を、短時間で触れるパズルに翻訳する
- 笑えるが、少しだけ胃が重い
- 説明を読まなくても、客タイプ名だけで状況がわかる
- 1シフトを遊ぶと「もう1回だけ」が成立する
- スクリーンショットで結果や盤面を共有したくなる

### 2.3 トーン

シニカル7、コメディ3。  
画面はクールで読みやすく、テキストだけにコラム的な乾いた一人称を残します。

## 3. ターゲット

- コンカフェコラムの既存読者
- コンカフェ文化に興味があるが、実店舗には詳しくない層
- カジュアルな落ちものパズルを短時間で遊びたい層

## 5. ゲームルール

### 5.1 基本操作

- 左右キー: 配置列を選択
- `Space` / `Enter` / `↓`: 来店中の客を配置
- 盤面タップ: その列に来店中の客を配置
- `R`: リスタート

### 5.2 1シフトの流れ

1. シフト開始
2. 来店中の客と次客が表示される
3. プレイヤーがキャスト列を選ぶ
4. 客が配置される
5. 各客の満足ターンが進む
6. ターンが0になった客は退店し、売上が加算される
7. 連続処理でボトル / シャンパンが発生する
8. 残り時間、出禁、盤面上限のいずれかで終了する

### 5.3 落下圧

来店中ゲージが0になると、現在選択中の列に自動配置されます。空気が悪いほど猶予時間が短くなります。

目的:

- 完全なターン制ではなく、営業中の焦りを出す
- 空気ゲージを単なるスコア補正ではなく、操作圧に変える
- 迷いすぎると事故る構造を作る

## 6. パズル盤面

### 6.1 盤面構成

- 列数: 5
- 行数: 9
- 各列の下にキャストが1人ずつ配置される
- 客は選択列の一番下の空きマスに配置される
- 満足して退店した客は盤面から消える
- 空きができると上の客が下へ詰まる

### 6.2 失敗条件

- 盤面上限到達
- クレーマー放置による出禁カウントが3に到達
- シフト時間終了

現プロトタイプでは、時間終了も営業終了として扱います。将来的には「時間切れ評価」と「ゲームオーバー」を分ける余地があります。

## 7. キャスト仕様

| キャスト | 役割 | 現在の効果 |
| --- | --- | --- |
| 営業 | 太客処理が得意 | 太客の処理ターン短縮、追加売上 |
| ガチ恋 | 高速処理の受け皿 | ガチ恋客が推し一致だと即帰宅に近い |
| 新人 | 低速だが個性枠 | 処理ターンが少し長い |
| 元ドル | 空気回復役 | 退店時に空気が少し回復 |
| ベテラン | VIP対応 | VIPの処理ターン短縮、追加売上 |

### 7.1 将来拡張

- シフト前のキャスト入れ替え
- 並び順によるシナジー
- 中央 / 端など位置依存スキル
- キャスト固有のボトル演出

## 8. 客仕様

| 客 | 基本挙動 | リスク | リターン |
| --- | --- | --- | --- |
| 太客 | 推し一致で強い | 処理がやや重い | 高売上 |
| 薄客 | 低リスク | 売上が低い | 場つなぎ |
| ガチ恋 | 推し一致で高速 | 推し被りが重い | 高めの売上 |
| 浮気性 | どの列でも推し扱い | 売上は低め | 配置しやすい |
| クレーマー | 放置で出禁カウント | 空気を大きく削る | 処理すると空気回復 |
| VIP | 指名キャストがある | 指名外配置が重い | 最高売上 |
| 同担拒否 | 隣の推し被りに弱い | ターン増加 | 配置パズル要素 |

## 9. 空気ゲージ

### 9.1 役割

空気ゲージは営業全体のムードです。高いほど処理が速く、低いほど来店圧が厳しくなります。

### 9.2 上昇要因

- 推し一致配置
- 太客処理
- クレーマー処理
- ガチ恋の推し一致処理
- 元ドル列での処理
- ボトル入り
- シャンパンコール

### 9.3 下降要因

- 推し不一致配置
- VIP指名外配置
- 推し被り
- クレーマー放置
- 時間経過による微減

### 9.4 レビュー観点

- 空気が高い時の気持ちよさが十分か
- 空気が低い時の焦りが理不尽ではないか
- 空気ゲージが「見るべき情報」として成立しているか

## 10. 連鎖代替: ボトル / シャンパン

### 10.1 ボトル入り

同じキャスト列で3人連続処理すると発生します。

効果:

- 空気が上昇
- その列の処理が少し進む
- サイドの営業記録に表示される

### 10.2 シャンパンコール

同じキャスト列で5人連続処理すると発生します。

効果:

- 空気が大きく上昇
- 各列から1人ずつ消える
- サイドの営業記録に表示される

### 10.3 課題

現状は「狙う」より「起きる」寄りです。今後は、列ごとの進捗、消えそうな客、次客との相性をより見やすくして、プレイヤーが意図的にコールを仕込めるようにします。

## 11. 常連化

### 11.1 現在の仕様

退店時に一定確率で客が常連化し、サイドパネルに表示されます。

### 11.2 将来仕様

- 常連は次シフトにも登場する
- 常連は推し固定
- 常連ごとの癖を覚える
- 雑に扱うと卒業する
- デイリーシードとは別に、自分だけの客層が育つ

## 12. UI仕様

### 12.1 画面構成

- 上部: タイトル、リスタート
- HUD: 空気、売上、残り時間、出禁
- 来店キュー: 来店中、次客、状況メッセージ
- FLOOR: 盤面とキャスト列
- 操作ボタン: 左、配置、右
- BACKROOM: シフトメモ、コール履歴、常連化

### 12.2 デザイン方針

- 顔は描かず、シルエットと色で人物を表現する
- 客は記号、キャストは人として扱う
- 暗い店内感は出すが、盤面の可読性を優先する
- ネオン過多にはしない
- スマホでも盤面、来店中、空気が一目で読めることを優先する

## 14. レビューで見てほしいこと

### 14.1 最優先

- 初見で何をするゲームか伝わるか
- 客とキャストの関係性がパズル判断になっているか
- 「推し一致」「推し被り」「指名外」が直感的にわかるか
- 空気ゲージが面白さに貢献しているか
- 1シフトのテンポが気持ちよいか

### 14.2 次点

- UIの情報量が多すぎないか
- スマホで遊べそうか
- 売上スコアの桁感が共有向きか
- コラム読者に刺さるトーンになっているか
- 客タイプやキャスト名に足したいものがあるか

### 14.3 今回は評価対象外でもよいもの

- 厳密なゲームバランス
- アニメーションの豪華さ
- 音の気持ちよさ
- ランキング
- 保存機能
- 長期育成

## 15. 既知の課題

- 連鎖を狙う戦略性がまだ弱い
- 常連化がメタ進行としてまだ機能していない
- 客の推し情報は出ているが、盤面全体の危険度はまだ読みづらい
- シャンパンコールの演出が弱い
- ゲーム終了条件が少し雑にまとまっている
- チュートリアルがない
- GitHub Pages公開前のため、外部レビューにはローカル起動かリポジトリ確認が必要
